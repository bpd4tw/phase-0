Week 1

1/22/2016

After completing the challenges in this first week of Phase 0, I've come to recognize the importance of version control for developers. The ability to collaborate on a project in unison not only optimizes time spent working, but also fosters discussions and modifications to create a better final project. Git "commits" provide invaluable "snapshots" throughout the course of a project's life. As a result, workers can always jump back to a previous version of code for any reason (i.e. debugging). Lastly, version control is a great tool for backing up work. Because GitHub is a remote site, committed work will still be accessible even if the local file is lost.

Let's delve a bit more into the git methods that keep track of changes: staging and committing. When a file is "staged (through the command 'git add')," git recognizes the changes made to that work/code. If a developer were to use the command "git status" in the command line, he/she would receive feedback regarding which files are ready to be committed and which files have been added (staged). Files are then committed through the command "git commit." The code and its changes have now been saved as a "snapshot" in this project's timeline. This is one of the "major key" benefits git provides. If necessary, we can circle back and revise changes during a project's development. To check on commits, use "git log" to view the recent commits on a local device.

I've already touched on this some, but will repeat it one more time: Using GitHub to store code is useful for both collaboration and storage. GitHub has a large list of users and is a fantastic environment for sharing and working on code with other developers. It can be used as a place to network, a site for working in concert or even as a resource for learning/checking out new code. The storage component is vital, too. As long as a user properly performs push/pull commands, he/she will always have two places where code exits:locally on an operating device and remotely on GitHub. So, in case of a computer crash or random coffee spill, all tracks are covered and no code is lost.
